# DVS2 document.rs.in Pattern Review

## Overview

Both DVS2 and miniextendr use a `document.rs.in` template that gets processed by configure to generate `document.rs`. This binary generates R wrapper code at build time.

## DVS2's document.rs.in

```rust
use @CARGO_STATICLIB_NAME@::R_WRAPPERS_DEPS_@PACKAGE_TARNAME_RS_UPPERCASE@ as DEPS;
use @CARGO_STATICLIB_NAME@::R_WRAPPERS_IMPLS_@PACKAGE_TARNAME_RS_UPPERCASE@ as IMPLS;
use @CARGO_STATICLIB_NAME@::R_WRAPPERS_IMPL_DEPS_@PACKAGE_TARNAME_RS_UPPERCASE@ as IMPL_DEPS;
use @CARGO_STATICLIB_NAME@::R_WRAPPERS_PARTS_@PACKAGE_TARNAME_RS_UPPERCASE@ as PARTS;

fn main() {
    // ... writes to "@PACKAGE_TARNAME@-wrappers.R"
}
```

## Key Design Decisions

### 1. Output Filename Pattern

```rust
.open("@PACKAGE_TARNAME@-wrappers.R")
```

Uses `@PACKAGE_TARNAME@` (lowercase, hyphenated) for the output filename. This matches the R package tarname convention.

**Example:** Package "dvs" produces `dvs-wrappers.R`

### 2. Constant Naming Pattern

Uses `@PACKAGE_TARNAME_RS_UPPERCASE@` for Rust constant names:
- `dvs` -> `DVS`
- `my-pkg` -> `MY_PKG`

This ensures valid Rust identifiers for any package name.

### 3. Header Comments

```rust
f.write_all(
    b"# ---- AUTO-GENERATED FILE - DO NOT EDIT ----
# This file is generated by the miniextendr proc-macro during package build.
# Any manual changes will be overwritten.
#
# To regenerate: rebuild the package.
# nolint start
# nocov start

",
)
```

Includes:
- Clear "do not edit" warning
- Generation context
- `nolint` and `nocov` guards for lintr/covr

### 4. Deduplication Logic

```rust
let mut seen = std::collections::HashSet::<&str>::new();
for group in roots {
    for &s in group {
        if seen.insert(s) {
            // ... write
        }
    }
}
```

Prevents duplicate R code from being written.

## Comparison with Miniextendr

Both projects use the same pattern. The key substitution variables are:

| Variable | Purpose | Example |
|----------|---------|---------|
| `@CARGO_STATICLIB_NAME@` | Rust crate name (underscores) | `dvs_rpkg` |
| `@PACKAGE_TARNAME@` | R package tarname (lowercase) | `dvs` |
| `@PACKAGE_TARNAME_RS_UPPERCASE@` | Rust constant suffix | `DVS` |

## Recommendation

The pattern is well-designed and shared between projects. No changes needed, but document this pattern clearly for downstream users creating their own miniextendr-based packages.
