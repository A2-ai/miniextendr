# Follow-ups from PR #382 review (`refactor/raise-condition-helper`)

PR #382 hoists the per-wrapper `switch(.val$kind, ...)` block into a single
`.miniextendr_raise_condition` helper emitted at the top of the generated
wrappers file. Net result: `rpkg/R/miniextendr-wrappers.R` shrinks 43,480 →
24,658 lines (-43%).

This plan tracks the loose ends identified during the self-review of #382.
Items are flat, in priority order. Items marked **skip** record why a
seemingly-obvious fix is wrong, so future readers don't redo the analysis.

## Skipped (analysis recorded so we don't redo it)

### Roxygen2 7.3.3 .Rd drift — **skip**
DESCRIPTION pins `Config/roxygen2/version: 8.0.0`. Local roxygen2 7.3.3 is
*older* than the project version, so its output is backward drift that would
break CI. Resolution is "upgrade local roxygen2 to 8.0.0 and re-run
`just devtools-document`," not a PR. Recorded in MEMORY.md
(`feedback_roxygen_version_drift.md`).

### `Cargo.lock` checksum churn — **skip**
Regenerated by every cargo build outside tarball mode. Build state, not a
fix target.

### Folding `inherits(.val, "rust_error_value")` guard into the helper — **skip**
Would add an R-level function call to every `.Call()` including the success
path. The current split (cheap inline check, dispatch in helper) is the
right cost balance.

### Symbol collision risk on `.miniextendr_raise_condition` — **accept**
Distinctive prefix; users are unlikely to define a function with the same
name. Future renames are wrappers-only changes since the symbol isn't
exported.
