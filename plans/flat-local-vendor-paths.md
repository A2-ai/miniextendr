# Flat local-crate vendor paths (hybrid layout)

Issue #284. Option 2: `cargo-revendor --freeze` writes flat `vendor/<name>/` paths for local workspace crates (miniextendr-api, miniextendr-macros, miniextendr-lint, etc.) while keeping `cargo vendor --versioned-dirs` for transitive deps. minirextendr already expects this layout on both the read and write side â€” only cargo-revendor drifted.

## Scope (flat priority)

### 1. `cargo-revendor/src/vendor.rs::extract_crate_archive`
Drop `pkg_version: Option<&str>`. This function is only called for local crates (single-version by construction); they always land at flat `vendor/<name>/`. Also extend the "remove existing dir" cleanup to nuke both the flat and versioned placeholder dirs â€” `cargo vendor --versioned-dirs` creates `vendor/<name>-<version>/` for local crates referenced via `[patch.crates-io]`, and that placeholder must not survive.

### 2. `cargo-revendor/src/vendor.rs::rewrite_local_path_deps`
Drop `versioned_dirs` parameter. For local crates, always emit `path = "../<name>"`. Intra-workspace refs (e.g., `miniextendr-api` depending on `miniextendr-macros`) point at flat sibling dirs.

### 3. `cargo-revendor/src/vendor.rs::freeze_manifest`
Split `patched_names` into local vs. transitive at write time:
- Name is in `local_pkgs` â†’ `path = "<rel>/<name>"` (flat).
- Name is only in `[patch.*]` â†’ existing `vendor_dir_name_for_pkg(...)` probe (versioned-first, flat fallback).

### 4. `cargo-revendor/src/main.rs::bootstrap_vendor_from_source_root`
Seed at flat `vendor/<name>/`, not `<name>-<version>`. The downstream `rewrite_local_path_deps` call on line 288 inherits the change (no separate `versioned_dirs` flag to pass).

### 5. `cargo-revendor/src/main.rs` (line 418-424)
`extract_crate_archive` call loses its `pkg_version` branch. Always flat for local.

### 6. Cargo vendor placeholder cleanup
After `run_cargo_vendor --versioned-dirs`, iterate `local_pkgs` and `std::fs::remove_dir_all(vendor_dir.join(format!("{}-{}", name, version)))` for each. `extract_crate_archive` then writes to `vendor/<name>/`. Covers the case where cargo vendor creates a versioned placeholder AND the `--flat-dirs` fallback where it creates a flat one (remove both pre-emptively).

### 7. `cargo-revendor/tests/multi_workspace.rs`
Existing `versioned_dirs_default_produces_versioned_layout` asserts `vendor/<local>-<version>/` exists. Flip to: for local, assert flat `vendor/<name>/` exists AND versioned does not; for transitive, keep versioned assertion. `flat_dirs_flag_produces_flat_layout` stays (everything flat under --flat-dirs).

### 8. `rpkg/src/rust/Cargo.toml`
Regenerated by `just vendor --freeze`. After change 3, `[dependencies]` and `[patch.crates-io]` both lose the version suffix for local crates.

### 9. `rpkg/src/rust/Cargo.lock`
Regenerated. May shift checksums/paths; commit if changed.

### 10. `rpkg/inst/vendor.tar.xz`
Gitignored â€” CI regenerates. No commit.

### 11. `minirextendr`
No code changes needed â€” readers already glob both layouts, writers already emit flat. Run `just minirextendr-test` to confirm (expect `test-scaffold-smoke.R` to stay green since it already asserts flat).

## Non-goals
- `--flat-dirs` behavior (unchanged; everything flat, reopens #214 by design â€” escape hatch).
- `cargo vendor` default-mode (--versioned-dirs stays for transitive).
- minirextendr scaffolding (already hybrid-aware).

## Risks
- `[patch.crates-io]` in rpkg/src/rust/Cargo.toml loses its version pin for local crates. This is fine â€” `path =` entries ignore `version =` anyway, and the local crate's own Cargo.toml carries the authoritative version.
- Mixed-layout `vendor/` tree may confuse external tooling. Acceptable trade for readable Cargo.toml.
