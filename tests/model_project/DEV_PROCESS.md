# Building a miniextendr R Package from Scratch

This document records every shell command and R/Rust invocation needed to go
from zero to a working R package (`pigworld`) backed by Rust, using the
miniextendr framework.

The working directory is the miniextendr monorepo root:
`/Users/elea/Documents/GitHub/miniextendr`

---

## Prerequisites

- R >= 4.5 with `devtools`, `usethis`, `withr`, `roxygen2` installed
- Rust >= 1.85 (`rustup` toolchain)
- `autoconf` (for generating `configure` from `configure.ac`)
- The miniextendr repo cloned locally

---

## Phase 1: Install the scaffolding helper

minirextendr is a pure-R package that ships templates and helpers for creating
new miniextendr projects.

```bash
R CMD INSTALL minirextendr
```

This installs the `minirextendr` package into your R library, making
`minirextendr::use_miniextendr()` and friends available.

---

## Phase 2: Create the R package skeleton

### 2a. Create directory and package structure with usethis

```r
Rscript -e '
library(minirextendr)

# usethis::create_package() creates DESCRIPTION, NAMESPACE, R/ directory.
# The "fields" argument lets us override the package name (which otherwise
# comes from the directory basename).
usethis::create_package(
  "tests/model_project",
  fields = list(Package = "pigworld"),
  open = FALSE,
  rstudio = FALSE
)
'
```

**What this produces:**
```
tests/model_project/
  DESCRIPTION       # Package = pigworld, Version = 0.0.0.9000
  NAMESPACE         # empty, will be regenerated by roxygen2
  R/                # empty directory for R source files
```

**Key detail:** `usethis::create_package()` validates the package name against
CRAN rules (ASCII letters, numbers, dots only; no underscores). The directory
name (`model_project`) doesn't need to match the package name (`pigworld`).

### 2b. Add miniextendr scaffolding

```r
Rscript -e '
library(minirextendr)

# Set the active usethis project to our package directory
usethis::proj_set("/Users/elea/Documents/GitHub/miniextendr/tests/model_project")

# use_miniextendr() adds all Rust build scaffolding.
# template_type = "rpkg" forces standalone-package mode (vs monorepo detection).
# local_path vendors miniextendr crates from the local checkout instead of GitHub.
use_miniextendr(
  template_type = "rpkg",
  local_path = "/Users/elea/Documents/GitHub/miniextendr"
)
'
```

**Why `template_type = "rpkg"`:** By default, `use_miniextendr()` auto-detects
the project type. Because we are running inside the miniextendr monorepo (which
has a `Cargo.toml` at its root), it detects "monorepo" and tries to create an
`rpkg/` subdirectory. Forcing `"rpkg"` tells it we want a standalone R package.

**Why `usethis::proj_set()` first:** `use_miniextendr()` writes files relative
to the active usethis project. Without explicitly setting it, usethis might
pick up the wrong project root.

**What this produces (new files and modifications):**

```
tests/model_project/
  DESCRIPTION              # Updated: added SystemRequirements, Config/build/* fields
  R/
    pigworld-package.R     # Package-level roxygen2 doc with useDynLib directive
  configure.ac             # autoconf template for build system detection
  configure.win            # Windows configure script
  configure.ucrt           # UCRT Windows configure script
  bootstrap.R              # R-level bootstrap for Config/build/bootstrap
  cleanup                  # Post-install cleanup script (Unix)
  cleanup.win              # Post-install cleanup script (Windows)
  cleanup.ucrt             # Post-install cleanup script (UCRT Windows)
  .Rbuildignore            # Patterns to exclude from R CMD build
  .gitignore               # Patterns to exclude from git
  tools/
    config.guess           # autoconf helper (platform detection)
    config.sub             # autoconf helper (platform canonicalization)
  inst/
    include/
      mx_abi.h             # C header for cross-package trait dispatch
  src/
    Makevars.in            # Makefile template (processed by configure)
    entrypoint.c.in        # C entry point template (DLL init, panic hooks)
    mx_abi.c.in            # C ABI for cross-package dispatch template
    win.def.in             # Windows DLL exports template
    vendor/                # Vendored miniextendr crates (copied from local_path):
      miniextendr-api/
      miniextendr-macros/
      miniextendr-macros-core/
      miniextendr-lint/
      miniextendr-engine/
    rust/
      Cargo.toml           # Rust project manifest (name = "pigworld")
      lib.rs               # Starter Rust code (hello + add examples)
      build.rs             # Build script (runs miniextendr-lint)
      document.rs.in       # R wrapper generator template
      cargo-config.toml.in # Cargo config template (for CRAN vendored builds)
```

## Phase 3: Configure Cargo.toml for local development

The scaffolded `Cargo.toml` points dependencies to GitHub:

```toml
[dependencies]
miniextendr-api = { git = "https://github.com/CGMossa/miniextendr" }

[build-dependencies]
miniextendr-lint = { git = "https://github.com/CGMossa/miniextendr" }
```

For local development inside the monorepo, add a `[patch]` section so Cargo
resolves these to the local workspace crates instead of fetching from GitHub:

```toml
# Appended to src/rust/Cargo.toml
[patch."https://github.com/CGMossa/miniextendr"]
miniextendr-api = { path = "../../../../miniextendr-api" }
miniextendr-macros = { path = "../../../../miniextendr-macros" }
miniextendr-macros-core = { path = "../../../../miniextendr-macros-core" }
miniextendr-lint = { path = "../../../../miniextendr-lint" }
```

The relative paths go from `tests/model_project/src/rust/` up four levels to
the monorepo root where the workspace crates live.

**Note:** `miniextendr-macros` and `miniextendr-macros-core` are transitive
dependencies of `miniextendr-api` -- they must also be patched or Cargo will
try to fetch them from GitHub.

## Phase 4: Generate the configure script

```bash
cd tests/model_project && autoconf
```

This processes `configure.ac` (the autoconf template) and produces the
executable `configure` script. The configure script detects `cargo`, `rustc`,
R paths, and generates:
- `src/Makevars` from `src/Makevars.in`
- `src/entrypoint.c` from `src/entrypoint.c.in`
- `src/mx_abi.c` from `src/mx_abi.c.in`
- `src/rust/.cargo/config.toml` from `src/rust/cargo-config.toml.in`
- `src/rust/document.rs` from `src/rust/document.rs.in`

---

## Phase 5: Run configure in development mode

```bash
cd tests/model_project && NOT_CRAN=true ./configure
```

**What `NOT_CRAN=true` does:**
- Skips vendoring (Cargo resolves deps via `[patch]` in Cargo.toml)
- Removes the generated `.cargo/config.toml` so Cargo uses normal resolution
- Cleans up stale vendor artifacts

**What configure does (in dev mode):**
1. Detects `cargo`, `rustc`, `sed`, platform info
2. Generates all build files from `.in` templates
3. Generates `Cargo.lock` (runs `cargo generate-lockfile`)
4. Runs `cargo vendor` to populate `src/vendor/` with external crates (not miniextendr crates -- those come from `[patch]`)
5. Creates `inst/vendor.tar.xz` (compressed vendor archive for CRAN)
6. Extracts the static library name from Cargo.toml (`CARGO_STATICLIB_NAME = pigworld`)

## Phase 6: Write the Rust model

Edit `tests/model_project/src/rust/lib.rs` -- replace the starter code with
the actual model. Key structural requirements:

```rust
use miniextendr_api::{miniextendr, miniextendr_module};

// The struct that will be exposed to R as an ExternalPtr.
// #[derive(miniextendr_api::ExternalPtr)] implements the TypedExternal trait,
// which is required for wrapping in R's external pointer type.
#[derive(miniextendr_api::ExternalPtr)]
pub struct World {
    // fields...
}

// #[miniextendr] on the impl block generates C wrappers and R wrapper metadata
// for each pub method.
#[miniextendr]
impl World {
    pub fn new(/* params */) -> Self { /* ... */ }
    pub fn step(&mut self) { /* ... */ }
    // ... more methods
}

// Register the module -- tells miniextendr which items to expose to R.
// "impl World" registers all #[miniextendr] methods from the impl block.
miniextendr_module! {
    mod pigworld;     // must match the crate name
    impl World;
}
```

**Critical:** `#[derive(miniextendr_api::ExternalPtr)]` on the struct is
mandatory. Without it, you get:
- `the trait bound 'World: TypedExternal' is not satisfied`
- `cannot find value 'R_WRAPPERS_RDATA_WORLD' in this scope`
- `cannot find value 'RDATA_CALL_DEFS_WORLD' in this scope`

## Phase 7: Verify the Rust code compiles

```bash
cd tests/model_project/src/rust && cargo check
```

This runs type checking without producing binaries. Catches syntax errors,
missing traits, type mismatches, etc. Much faster than a full build.

---

## Phase 8: Build and install the R package (first pass)

```bash
NOT_CRAN=true R CMD INSTALL tests/model_project
```

This is the full R package installation pipeline:
1. R calls `./configure` (re-generates build files)
2. Compiles C files (`entrypoint.c`, `mx_abi.c`) with the system C compiler
3. Runs `cargo build --release` to compile Rust into a static library (`libpigworld.a`)
4. Runs the `document` binary (from `document.rs`) to generate R wrapper source
5. Links everything into `pigworld.so` (the shared library R loads)
6. Installs to the R library (`~/Library/R/arm64/4.5/library/pigworld/`)

After this step, `library(pigworld)` works but the R wrappers in `R/` are
still the stale scaffolded versions (just `hello` and `add`).

---

## Phase 9: Generate R wrappers with devtools::document()

```r
Rscript -e 'devtools::document("tests/model_project")'
```

**What this does:**
1. Loads the package (triggers recompilation if needed)
2. Runs the `document` binary -- this is a Rust program that reads the
   `#[miniextendr]` proc-macro output and generates `R/pigworld-wrappers.R`
3. Runs `roxygen2::roxygenize()` to:
   - Parse roxygen2 comments in `R/*.R` files
   - Regenerate `NAMESPACE` with proper `export()` and `useDynLib()` directives
   - Generate `man/*.Rd` help files

**What gets generated/updated:**
- `R/pigworld-wrappers.R` -- R class definition for `World` with all methods
  (`$new`, `$step`, `$run`, `$get_tick`, `$population`, `$x_positions`,
  `$y_positions`, `$energies`, `$ages`, `$summary`) plus S3 method dispatch
  (`$.World`, `[[.World`)
- `NAMESPACE` -- exports `World`, registers S3 methods, `useDynLib(pigworld)`
- `man/World.Rd` -- help page for the World class
- `man/pigworld-package.Rd` -- package help page

---

## Phase 10: Rebuild with generated wrappers (second pass)

```bash
NOT_CRAN=true R CMD INSTALL tests/model_project
```

The first install compiled the Rust code but the R wrappers were stale. Now
that `devtools::document()` has regenerated them, this second install
incorporates the correct R code. After this, the package is fully functional.

**Why two installs?** The R wrappers are generated by running the compiled Rust
`document` binary. But that binary can only exist after the first compilation.
So the cycle is: build (compiles Rust) -> document (generates R wrappers) ->
build again (includes correct R wrappers).

---

## Phase 11: Test from R

```r
Rscript -e '
library(pigworld)

w <- World$new(20L, 3.0, 1.0, 80.0, 100L, 5.0)
cat(w$summary(), "\n")
# => Tick 0: 20 pigs, avg energy 50.0

w$run(50L)
cat(w$summary(), "\n")
# => Tick 50: 14 pigs, avg energy 36.2

cat("Population:", w$population(), "\n")
cat("X positions:", head(w$x_positions(), 5), "\n")
cat("Y positions:", head(w$y_positions(), 5), "\n")
cat("Energies:", head(w$energies(), 5), "\n")
cat("Ages:", head(w$ages(), 5), "\n")

w$run(50L)
cat(w$summary(), "\n")
# => Tick 100: 5 pigs, avg energy 24.6
'
```

## Summary: the minimal command sequence

For reference, here is the shortest path from zero to a working package,
assuming the miniextendr repo is at `/path/to/miniextendr`:

```bash
# 1. Install the scaffolding tool
R CMD INSTALL minirextendr

# 2. Create R package + add miniextendr scaffolding
Rscript -e '
library(minirextendr)
usethis::create_package(
  "tests/model_project",
  fields = list(Package = "pigworld"),
  open = FALSE,
  rstudio = FALSE
)
usethis::proj_set("tests/model_project")
use_miniextendr(
  template_type = "rpkg",
  local_path = "/path/to/miniextendr"
)
'

# 3. (Manual) Add [patch] section to tests/model_project/src/rust/Cargo.toml
#    for local development (see Phase 3 above)

# 4. (Manual) Edit tests/model_project/src/rust/lib.rs with your Rust code
#    Key: #[derive(miniextendr_api::ExternalPtr)] on structs exposed to R

# 5. Generate configure script
cd tests/model_project && autoconf

# 6. Configure (generates Makevars, Cargo.lock, vendor archive)
NOT_CRAN=true ./configure

# 7. Verify Rust compiles
cd src/rust && cargo check && cd ../..

# 8. First install (compiles Rust, installs package)
NOT_CRAN=true R CMD INSTALL tests/model_project

# 9. Generate R wrappers from compiled Rust
Rscript -e 'devtools::document("tests/model_project")'

# 10. Second install (incorporates generated R wrappers)
NOT_CRAN=true R CMD INSTALL tests/model_project

# 11. Test
Rscript -e 'library(pigworld); w <- World$new(20L, 3.0, 1.0, 80.0, 100L, 5.0); cat(w$summary())'
```

## Appendix: R packages/functions used

| Package | Function | Purpose |
|---------|----------|---------|
| `usethis` | `create_package()` | Creates DESCRIPTION, NAMESPACE, R/ directory |
| `usethis` | `proj_set()` | Sets the active project directory for subsequent usethis calls |
| `minirextendr` | `use_miniextendr()` | Adds all miniextendr scaffolding (configure.ac, Makevars.in, Cargo.toml, lib.rs, build.rs, entrypoint.c.in, etc.) |
| `devtools` | `document()` | Loads package, runs document binary, runs roxygen2 to generate R wrappers + NAMESPACE + man pages |

## Appendix: Rust macros/derives used

| Item | Purpose |
|------|---------|
| `#[derive(miniextendr_api::ExternalPtr)]` | Implements `TypedExternal` trait on a struct, enabling it to be wrapped in R's `EXTPTRSXP` |
| `#[miniextendr]` on `impl` block | Generates C wrapper functions and R wrapper metadata for each `pub` method |
| `miniextendr_module! { mod name; impl Type; }` | Registers all exported items with R's `.Call` interface |

## Appendix: Shell tools used

| Tool | Purpose |
|------|---------|
| `autoconf` | Generates `configure` from `configure.ac` |
| `R CMD INSTALL` | Full R package build + install pipeline |
| `cargo check` | Rust type-checking without full compilation |
| `cargo vendor` | (Called by configure) vendors external crate dependencies |
