# R Package Entry Point (`R_init_*`)

This document explains the initialization requirements for R packages using
miniextendr. The entry point is the `R_init_<pkgname>()` function in
`entrypoint.c`.

## Overview

When R loads a package's shared library, it calls `R_init_<pkgname>()`. This
function must initialize miniextendr's runtime before any Rust code runs.

## Required Initialization Order

```c
void R_init_mypkg(DllInfo *dll) {
    // 1. Install panic hook (must be first)
    miniextendr_panic_hook();

    // 2. Initialize worker thread (must be before any R API calls from Rust)
    miniextendr_worker_init();

    // 3. Register trait ABI C-callables (if using cross-package traits)
    mx_abi_register();

    // 4. Register R functions (generated by miniextendr_module!)
    R_init_mypkg_miniextendr(dll);

    // 5. Lock down symbol visibility
    R_useDynamicSymbols(dll, FALSE);
    R_forceSymbols(dll, TRUE);
}
```

## Functions Explained

### `miniextendr_panic_hook()`

Installs a custom panic hook that captures panic messages for R error reporting.
Without this, panics show generic "unknown panic" messages.

**Must be called first** - before any Rust code that might panic.

### `miniextendr_worker_init()`

Initializes the worker thread infrastructure:
- Records the main thread ID for thread safety checks
- Spawns the worker thread for `run_on_worker()`
- Sets up channels for bidirectional communication

**Must be called from R's main thread.** Calling from another thread causes
incorrect thread detection for all subsequent R API calls.

**Must be called before any R API calls from Rust.** Functions like
`Rf_allocVector` will panic if called before initialization.

### `mx_abi_register()`

Registers C-callable functions for the trait ABI:
- `mx_wrap` - Wrap a Rust value in an R external pointer
- `mx_get` - Get the base vtable from an external pointer
- `mx_query` - Query for a trait vtable by tag

This is **required if** your package:
- Defines traits with `#[miniextendr]`
- Implements traits with `#[miniextendr] impl Trait for Type`
- Uses cross-package trait dispatch

Consumer packages load these via `R_GetCCallable()` at runtime.

### `R_init_<pkgname>_miniextendr(dll)`

Generated by `miniextendr_module!`. Registers:
- All `#[miniextendr]` functions
- ALTREP classes
- Impl block methods

This function is defined in your Rust code, not `entrypoint.c`.

### `R_useDynamicSymbols(dll, FALSE)`

Prevents R from looking up symbols by name at runtime. All symbols must be
registered via `R_registerRoutines()` (which `R_init_*_miniextendr` does).

### `R_forceSymbols(dll, TRUE)`

Forces `.Call()` to use registered symbols only. This catches typos in
function names at load time rather than call time.

## Template Location

The entry point template is at `rpkg/src/entrypoint.c.in`. The configure script
substitutes `@PACKAGE_TARNAME_RS@` with the actual package name (with hyphens
converted to underscores for C identifier compatibility).

## When APIs Can Be Called

| API Category | When Safe |
|--------------|-----------|
| Panic hook | Anytime after `miniextendr_panic_hook()` |
| R APIs (value-returning) | After `miniextendr_worker_init()`, routed via `with_r_thread` |
| R APIs (pointer-returning) | Main thread only, after `miniextendr_worker_init()` |
| Trait ABI | After `mx_abi_register()` |
| User Rust functions | After `R_init_*_miniextendr(dll)` |

## Minimal Example

For a package named `myrust`:

```c
#include <R.h>
#include <Rinternals.h>
#include <R_ext/Rdynload.h>

extern void R_init_myrust_miniextendr(DllInfo *dll);
extern void miniextendr_panic_hook(void);
extern void miniextendr_worker_init(void);

void R_init_myrust(DllInfo *dll) {
    miniextendr_panic_hook();
    miniextendr_worker_init();
    R_init_myrust_miniextendr(dll);
    R_useDynamicSymbols(dll, FALSE);
    R_forceSymbols(dll, TRUE);
}
```

## Embedding R (miniextendr-engine)

When embedding R in a Rust application (not an R package), initialization
differs slightly:

```rust
use miniextendr_engine::REngine;

fn main() {
    // REngine::build() handles R initialization
    let _r = REngine::build().unwrap();

    // After this, you can call R APIs
    // miniextendr_worker_init() is called automatically
}
```

Additional functions available when embedding:
- `miniextendr_encoding_init()` - Initialize UTF-8 locale handling (non-API)

These aren't available in R packages because they reference symbols not
exported from libR.

## Troubleshooting

### "miniextendr_worker_init() must be called"

This panic means R API functions were called before initialization. Ensure
`miniextendr_worker_init()` is called in `R_init_*()` before any Rust code
that uses R APIs.

### Thread check failures

If `is_r_main_thread()` returns incorrect results, `miniextendr_worker_init()`
was likely called from the wrong thread. It **must** be called from R's main
thread (the thread that called `R_init_*()`).

### Symbol not found errors

Ensure all extern declarations in `entrypoint.c` match the Rust exports.
The functions are defined in miniextendr-api with `#[no_mangle]` or
`#[unsafe(no_mangle)]`.
