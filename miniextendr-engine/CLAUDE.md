# miniextendr-engine

Code-generation engine — whole-crate emission that walks distributed_slice registrations and produces wrapper output. See root `CLAUDE.md` for project rules.

## Scope
- Consumes the linkme `#[distributed_slice]` registrations declared in `miniextendr-api/src/registry.rs`.
- Produces R wrapper text written to `rpkg/R/miniextendr-wrappers.R` at install time (via `miniextendr_write_wrappers`).

## Layout
- `lib.rs` — single-file crate. `collect_r_wrappers()` lives here: deduplicates, priority-sorts (Sidecar=0, Class=1, Function=2, TraitImpl=3, Vctrs=4), topologically sorts S7 classes for parent-before-child ordering.
- `tests.rs` — collection/order/dedup tests.

## When to touch
- Adding a new `RWrapperPriority` variant or new distributed_slice → wire it in here.
- Changing R wrapper ordering rules — this is the only place that decides emission order. R-side `library()` evaluates definitions top-down, so S7-parent-before-child matters.

## Notes
- `document.rs` and `miniextendr_link_registrations()` were deleted 2026-03-09 — the old NAMESPACE-driven flow is gone. Wrappers are now generated by loading the cdylib, invoking `miniextendr_write_wrappers`, then re-linking as staticlib.
