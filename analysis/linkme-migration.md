# linkme Migration: Analysis and Evaluation

## Problem Statement

`miniextendr_module!` was a 221-line manual registration macro in `rpkg/src/rust/lib.rs`.
Users had to list every `fn`, `impl`, `struct`, `use`, and `impl Trait for Type` — duplicating
what `#[miniextendr]` already knew. In rpkg alone, the root module block contained 73 `use`
entries and 50 `fn` entries, and every child module (73 of them) had its own
`miniextendr_module!` block.

The entire `miniextendr-lint` crate (54 lint rules, 6 rule modules totaling 911 lines) existed
primarily to catch mismatches between `#[miniextendr]` attributes and the module declarations.

**Goal**: Replace manual registration with automatic link-time registration via linkme's
`#[distributed_slice]`, reducing user-facing boilerplate to zero module listings.

## Strategy Evaluation

Eleven strategies (A through K) were evaluated for solving the archive extraction problem.
The core challenge: when a Rust staticlib (`.a`) is linked into an R shared library (`.so`),
the linker only extracts `.o` members that resolve undefined symbols. With linkme, the
`#[distributed_slice]` entries live in user crate `.o` files, but nothing references them
directly — `miniextendr_register_routines()` reads the slices from miniextendr-api, not the
user crate.

### Strategies Tested

| # | Strategy | Verdict | Why |
|---|----------|---------|-----|
| A | `-Wl,-force_load` / `--whole-archive` | **CHOSEN** (for .so) | Simplest. Forces linker to extract all `.o` from archive. Platform-specific flags (macOS vs Linux) handled by configure. |
| B | Targeted `-force_load` via build.rs path | Not feasible | build.rs runs before compilation — archive doesn't exist yet. |
| C | `-Wl,-u,_symbol` to force extraction | Fragile | Requires knowing exact symbol names at Makevars time. |
| D | build.rs generates references to C wrappers | Not feasible | Circular dependency — symbols generated by proc macro, unavailable to build.rs. |
| E | Proc macro emits anchor symbol | Not feasible | Proc macros have no global state — can't emit "exactly once". |
| F | Build-time wrapper generation (build.rs) | Future option | Sound architecture but requires ~2000 lines of R codegen refactored out of proc macros. |
| G | cdylib instead of staticlib | Fights R | R's build system expects to control the final link step. |
| H | `#[used]` + `#[link_section]` | Doesn't work | These affect the compiler, not the linker. The `.o` file is never opened. |
| I | Fat LTO (single `.o` merging) | Too slow | Would work but LTO significantly increases compile time. Not viable for dev builds. |
| J | build.rs emits `-u` flags from scanned symbols | Promising but fragile | Symbol name prediction must exactly match macro output, including platform prefix. |
| K | Every `#[miniextendr]` emits anchor entry | Doesn't work | Archive extraction is a linker problem — more entries inside un-extracted `.o` files don't help. |

### Key Insight

The problem exists at two levels:

1. **`.a` → `.so` linking** (R package build): Solved by `-Wl,-force_load` in Makevars.in.
   The linker extracts ALL `.o` members from the archive, preserving linkme sections.

2. **rlib → binary linking** (document binary): Cargo's rlib linking doesn't support
   `-force_load`. Solved by generating an empty `#[no_mangle]` anchor function
   (`miniextendr_link_registrations()`) in the user crate that the binary calls, forcing
   the linker to pull in at least one `.o` file. Inter-module `mod` declarations then
   create a reference chain that pulls the rest.

## Architecture Chosen

### Registration Flow

```
Compile time:
  #[miniextendr] fn add(...)  →  #[distributed_slice(MX_CALL_DEFS)] static ...
                               →  #[distributed_slice(MX_R_WRAPPERS)] static ...

Package load (R_init_*):
  entrypoint.c
    → miniextendr_panic_hook()
    → miniextendr_runtime_init()
    → miniextendr_assert_utf8_locale()
    → miniextendr_set_altrep_pkg_name()
    → miniextendr_init_vctrs()
    → mx_abi_register()
    → miniextendr_register_routines(dll)   ← reads MX_CALL_DEFS, MX_ALTREP_REGISTRATIONS

Document generation (cargo run --bin document):
    → miniextendr_link_registrations()      ← anchor for rlib extraction
    → write_r_wrappers_to_file(path)        ← reads MX_R_WRAPPERS, sorts, writes R file
```

### Four Distributed Slices

| Slice | Entry Type | Purpose |
|-------|-----------|---------|
| `MX_CALL_DEFS` | `R_CallMethodDef` | `.Call` routine registration with R |
| `MX_R_WRAPPERS` | `RWrapperEntry` | R wrapper code fragments (with priority ordering) |
| `MX_ALTREP_REGISTRATIONS` | `fn()` | ALTREP class init functions |
| `MX_TRAIT_DISPATCH` | `TraitDispatchEntry` | `(concrete_tag, trait_tag) → vtable` mapping |

### R Wrapper Ordering

linkme doesn't guarantee cross-file ordering. `RWrapperEntry` carries a priority:

```
Sidecar(0) < Class(1) < Function(2) < TraitImpl(3) < Vctrs(4)
```

Within the Class priority, S7 classes are topologically sorted (parent before child) by
parsing `parent = ParentName` from the generated R code.

### Universal Query

Replaces per-type query functions with a single function that scans `MX_TRAIT_DISPATCH`:

```rust
pub unsafe extern "C" fn universal_query(
    ptr: *mut mx_erased, trait_tag: mx_tag,
) -> *const c_void {
    let concrete_tag = (*(*ptr).base).concrete_tag;
    for entry in MX_TRAIT_DISPATCH.iter() {
        if entry.concrete_tag == concrete_tag && entry.trait_tag == trait_tag {
            return entry.vtable;
        }
    }
    std::ptr::null()
}
```

Linear scan is appropriate — typical packages have <50 trait impls, and the array is
contiguous and cache-friendly.

## Platform Considerations

### macOS
- `.so` linking: `-Wl,-force_load,<path_to_archive>`
- Targets a single archive (safe — no risk of duplicate symbols from system libs)

### Linux
- `.so` linking: `-Wl,--whole-archive <archive> -Wl,--no-whole-archive`
- Must bracket with `--no-whole-archive` to avoid pulling everything from subsequent libs
- Currently using macOS flags only (configure should detect platform for cross-platform support)

### Windows
- Not yet tested. May need `/WHOLEARCHIVE:<archive>` (MSVC) or `--whole-archive` (MinGW).

## Linker Behavior Lessons

1. **Archive extraction is lazy**: The linker only opens `.o` files from a `.a` that resolve
   pending undefined symbols. No reference = no extraction = no linkme entries.

2. **`#[used]` is compiler-only**: It prevents the compiler from dead-stripping a symbol
   within a `.o` file, but the linker never sees that `.o` if nothing references it.

3. **`R_GetCCallable` throws, doesn't return NULL**: When a callable isn't registered (e.g.,
   because the providing package's DLL hasn't loaded), R issues `Rf_error()` (a longjmp).
   Our null-check after the call was unreachable dead code.

4. **NAMESPACE `importFrom` controls DLL load order**: `importFrom(vctrs, new_vctr)` in
   NAMESPACE forces R to load the vctrs namespace (and its DLL) before the importing
   package's DLL. Without these entries, `R_GetCCallable("vctrs", ...)` in `R_init_*` fails
   because vctrs hasn't registered its callables yet.

## Future Directions

### Eliminate the Document Binary (Strategy F)

The document binary exists solely to read `MX_R_WRAPPERS` at link time and write an R file.
This could be moved to build.rs by having the lint/engine crate generate R wrappers from
source parsing — the same parse tree the lint already builds. This would:

- Remove the `[[bin]]` section from Cargo.toml
- Remove the `document.rs` file
- Remove the need for `crate-type = ["staticlib", "lib"]` (just `staticlib`)
- Remove `miniextendr_link_registrations()` entirely

The R wrapper generation code (~2000 lines) would need to be extracted from the proc macro
crate into a shared library. This is architecturally sound but a significant refactoring effort.

### Cross-Platform Linker Flags

Currently hardcoded to macOS (`-Wl,-force_load`). The configure script should detect the
platform and emit the appropriate flags:

```makefile
# macOS
PKG_LIBS = -Wl,-force_load,$(CARGO_AR)
# Linux
PKG_LIBS = -Wl,--whole-archive $(CARGO_AR) -Wl,--no-whole-archive
# Windows (MSVC)
PKG_LIBS = /WHOLEARCHIVE:$(CARGO_AR)
```
