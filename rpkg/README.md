# Rust-based, R-package `rpkg`

## Build System Overview

This package uses a Rust backend with autoconf-based configuration. The build
system is designed to work both in the monorepo (for development) and as a
standalone package (for CRAN/installation).

## Files That Must Be Committed

The following generated files **must be committed** to version control:

- `configure` - Generated from `configure.ac` by running `autoconf`
- `R/miniextendr_wrappers.R` - R function wrappers generated by the `document` binary

These are committed because:

- `configure` - Users installing the package shouldn't need autoconf installed
- `R/miniextendr_wrappers.R` - R CMD build expects R/*.R files to exist

## Developer Guide

### Regenerating configure

When you modify `configure.ac`, regenerate `configure`:

```shell
cd rpkg && autoconf
```

### Regenerating R wrappers

When you modify Rust code that changes the R API (add/remove/rename exported
functions), regenerate the wrappers:

```shell
just load  # or devtools::load_all("rpkg")
```

This runs cargo build and the document binary, then copies the generated
`src/miniextendr_wrappers.R` to `R/miniextendr_wrappers.R`.

### Build Flow

1. **bootstrap.R** (runs before R CMD build via `Config/build/bootstrap: TRUE`):
   - Copies `miniextendr-api` and `miniextendr-macros` from monorepo to `inst/vendor/`
   - Patches workspace inheritance in Cargo.toml files
   - Runs `./configure`

2. **configure** (runs during R CMD INSTALL):
   - Sets `NOT_CRAN=true` if not already set
   - Generates `src/Makevars`, `src/rust/Cargo.toml`, `src/rust/.cargo/config.toml`

3. **Makevars** (runs during R CMD INSTALL):
   - Runs `cargo vendor` (only if `NOT_CRAN=true`)
   - Builds Rust code with `cargo build`
   - Generates R wrappers with `cargo run --bin document`
   - Links the shared library

### CRAN vs Development Builds

- **CRAN builds**: `NOT_CRAN` is not "true", so vendoring is skipped. The tarball
  must already contain all vendored dependencies in `inst/vendor/`.

- **Development builds**: `NOT_CRAN=true` (set by devtools or configure), so
  `cargo vendor` runs to refresh dependencies.
